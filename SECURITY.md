# Security Policy

## Supported Versions

Security fixes are backported to the current minor release only.

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| < 1.1   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public issue.

Instead, email the maintainer directly at **tonegabes@gmail.com** with:

- A short description of the vulnerability.
- Steps to reproduce (proof of concept if possible).
- The affected version(s).
- Any suggested mitigation.

You should receive an initial response within 72 hours. Please allow a reasonable
window for a fix before any public disclosure. Responsible disclosures will be
credited in the CHANGELOG unless you request otherwise.
